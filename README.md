# Copilot 코딩 에이전트로 팀 확장하기

_Copilot 코딩 에이전트가 GitHub에서 직접 이슈를 처리하도록 해보세요. 코딩 환경이 필요 없습니다._

## 환영합니다

- **대상**: 코드 워크플로우를 가속화하려는 모든 수준의 개발자.
- **학습 내용**: Copilot에 이슈를 할당하는 방법, Copilot의 작업을 리뷰하는 방법, Copilot에 피드백을 제공하는 방법, 여러 이슈를 병렬로 처리하는 방법을 배웁니다.
- **구축 내용**: Mergington High School 방과후 활동 웹사이트의 저장소에 GitHub Copilot 접근 권한을 부여하여, 동료 교사들이 직접 코딩하지 않고도 변경할 수 있도록 합니다.
- **사전 요구사항**:

  - Copilot 코딩 에이전트를 사용하려면 GitHub Copilot 구독(Pro 이상)이 필요합니다.
  - Skills 실습: [GitHub 소개](https://github.com/skills-kr/introduction-to-github)
  - Skills 실습: [GitHub Copilot 시작하기](https://github.com/skills-kr/getting-started-with-github-copilot)

- **소요 시간**: 이 실습은 완료하는 데 1시간 미만이 소요됩니다.

이 실습에서는 다음을 수행합니다:

1. 저장소에서 Copilot 코딩 에이전트를 활성화하는 방법을 배웁니다.
1. 이슈를 할당하고 에이전트의 진행 상황을 이해합니다.
1. 코드 변경에 대해 에이전트와 협업합니다.
1. 에이전트의 작업 환경을 커스터마이징하고 최적화합니다.
1. 여러 작업을 병렬로 위임하고 추적합니다.

### 이 실습을 시작하는 방법

실습을 여러분의 계정으로 복사한 후, 여러분이 좋아하는 Octocat(Mona)에게 **약 20초** 정도 첫 번째 레슨을 준비할 시간을 주세요. 그런 다음 **페이지를 새로고침**하세요.

실습을 복사할 때 다음 설정을 권장합니다:

- 소유자(owner)는 개인 계정이나 조직을 선택하세요.

- 공개 저장소 생성을 권장합니다. 비공개 저장소는 Actions 사용 시간을 소비합니다.

20초 후에도 실습이 준비되지 않으면 [Actions](../../actions) 탭을 확인해 주세요.

- 작업이 실행 중인지 확인하세요. 때때로 시간이 좀 더 걸릴 수 있습니다.

- 실패한 작업이 표시되면 이슈를 제출해 주세요. 멋지네요, 버그를 발견하셨습니다! 🐛
